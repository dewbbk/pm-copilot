# Review Request — Запрос ревью у стейкхолдера

> Copilot формирует запрос стейкхолдеру с нужным контекстом из ПРД и goal-файла (`~/pm-copilot/goals/[slug].md`). Разные аудитории — разная глубина и фокус. Загружается по команде `запросить ревью`.

## Когда использовать

- PM подготовил ПРД и хочет получить ревью от Tech Lead, аналитика, дизайнера или C-level
- PM хочет запросить фидбек на конкретной стадии (draft, in_review, ready)

## Аудитории и формат

- **Tech Lead**: Техническая реализация, зависимости, риски интеграции. Полный контекст: user stories, API, edge cases.
- **Аналитик**: Метрики, цепочка влияния, ганарды, методология оценки. Средний контекст.
- **Дизайнер**: User stories, сценарии, ограничения. Средний контекст.
- **C-level / PO**: Бизнес-эффект, сроки, риски, ask. Краткий контекст: Executive Summary.

## Шаблон

```markdown
# Запрос ревью: [Название задачи]

> PM: [имя] | Аудитория: [роль] | Дата: [дата] | Версия ПРД: [draft/in_review/ready]

## Что ревьюим
[1-2 предложения: что сделано и на каком этапе]

## Ключевые вопросы для ревью
[Сгенерированные под аудиторию]

## Контекст
[Адаптированный контекст из ПРД]

## Что ожидаем в ответе
- [ ] [Вопрос 1]
- [ ] [Вопрос 2]
- [ ] Общий комментарий / одобрение
```

## Генерация вопросов под аудиторию

- **Tech Lead**: «Реализуемы ли user stories за [X] спринтов?», «Зависимости от других команд?», «Технические риски?»
- **Аналитик**: «Корректна ли цепочка влияния?», «Достаточны ли ганарды?», «Реалистична ли оценка эффекта?»
- **Дизайнер**: «Покрывают ли user stories все сценарии?», «Ограничения, влияющие на UX?»
- **C-level / PO**: «Согласны с приоритетом и сроками?», «Стратегические риски?»

## Механика

1. PM выбирает аудиторию (или нескольких)
2. Copilot загружает goal-файл + epic-файлы + ПРД
3. Генерирует Review Request с адаптированным контекстом
4. PM проверяет и правит вопросы
5. Предлагает формат: Markdown / Confluence
6. Запись в goal-файл: секция `review_requests` в YAML frontmatter

## Goal-файл (структура review_requests)

```yaml
# ~/pm-copilot/goals/[slug].md — фрагмент frontmatter
review_requests:
  - audience: "Tech Lead"
    status: pending | responded | integrated
    sent_at: datetime
    questions: []
    response: string | null
    integrated_at: datetime | null
```
